# QA: Корзина тестирования

Плавающий виджет-«таблетка» для команды тестирования, который живёт поверх любой вкладки браузера и в реальном времени показывает, что стоит на проверке прямо сейчас. Реализован как userscript (Tampermonkey) с бэкендом на Google Apps Script.

**- [Живое демо](https://github.com/larina-ch-daria/Jira-tracker/index.html)**

> Демо работает на **вымышленных данных** — реальная Jira, токены и URL в него не входят.

## Зачем это

Тестировщики и тимлид тратили время на то, чтобы вручную заходить в Jira и проверять, не появились ли новые карточки на тестировании. Виджет убирает этот шаг: сводка всегда на экране, а о новых карточках сообщает звук — не нужно держать вкладку Jira открытой.

## Что умеет

- **Таблетка `QA: N`** в углу экрана — общее число карточек на тестировании. Цвет меняется по нагрузке: зелёный (пусто) → серый (норма) → оранжевый (≥10) → красный (≥30). Перетаскивается, позиция запоминается.
- **Панель со вкладками** — карточки сгруппированы по доскам, внутри — по эпикам. Отдельные вкладки на каждую доску и на очередь поддержки.
- **Вкладка «Саппорт»** — обращения, сгруппированные по тяжести (Критическая → Средняя → Незначительная → Не ошибка) с цветовой кодировкой и указанием дежурного.
- **Селектор тестировщиков** — переключение между несколькими watchlist'ами (напр. тимлид смотрит сразу несколько человек).
- **Звуковые оповещения о новых карточках** — у каждого тестировщика свой звук; два режима («обычный» — синтезированные сигналы, «эпичный» — кастомные аудиофайлы); индивидуальный mute; предпрослушка; дедупликация и очередь звуков для фоновых вкладок.
- **Подсветка новинок** — новые с прошлого просмотра карточки помечаются бейджем `NEW`, таблетка пульсирует; отметка о просмотре снимается автоматически.

## Архитектура

```
Google Apps Script (на аккаунте владельца)
   ├─ по расписанию читает Jira (REST API)
   ├─ раскладывает карточки по доскам/эпикам/тестировщикам в Google Sheets
   └─ doGet() отдаёт снапшот в виде JSON  ← endpoint Web App

Tampermonkey userscript (у каждого в браузере)
   └─ GET на URL Web App → рендерит таблетку и панель
```

Такое разделение выбрано намеренно: **Jira-токен лежит в одном месте** (Apps Script), а на устройствах пользователей — только URL эндпоинта. Отозвать доступ можно, сняв деплой Web App, не трогая токен.

## Технологии

Vanilla JS (без фреймворков и сборки), Web Audio API для синтезированных звуков, Pointer Events для drag'n'drop, Google Apps Script, Jira REST API, Google Sheets как хранилище.

## Демо в этом репозитории

`index.html` — самодостаточная страница, повторяющая интерфейс и логику боевого виджета, но на вымышленных данных и без каких-либо секретов. Открывается как обычный файл в браузере или публикуется через GitHub Pages. Кнопки слева внизу имитируют приход новой карточки, чтобы показать подсветку и звук.

## Запуск демо

Локально — просто открыть `index.html` в браузере.

Через GitHub Pages: **Settings → Pages → Source: `main` / root**. Файл называется `index.html`, поэтому демо будет доступно прямо по адресу репозитория `https://<username>.github.io/<repo>/`.
